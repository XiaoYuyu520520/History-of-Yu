怎么在这个开发板上转换onnx模型为om模型

好的，在您的 Atlas 200I DK A2 开发板上将 ONNX 模型转换为昇腾（Ascend）专用的 OM (Offline Model) 模型，核心是使用昇腾 CANN (Compute Architecture for Neural Networks) 工具包中提供的 atc (Ascend Tensor Compiler) 命令。

根据您之前提供的日志，您的环境已经基本就绪。下面是详细的步骤和说明。

核心工具：ATC 命令

atc 命令是模型转换的枢纽。它的作用是：

  1. 解析模型：读取 ONNX、TensorFlow、PyTorch 等框架的模型文件。

  2. 算子融合：将模型中的多个算子（Operator）融合成昇腾 NPU 支持的高性能算子。

  3. 优化图结构：对计算图进行优化，减少计算量和内存占用。

  4. 生成离线模型：编译生成高度优化、适配昇腾 NPU 硬件的 .om 文件。

转换步骤

第 1 步：准备环境

在执行 atc 命令之前，您需要先加载 CANN 工具包的环境变量。这通常是通过执行一个脚本来完成的。默认安装路径下，该脚本位于 /usr/local/Ascend/ascend-toolkit/set_env.sh。

在您的终端中运行：

bash
复制
source /usr/local/Ascend/ascend-toolkit/set_env.sh

提示: 您可以把这行命令添加到 ~/.bashrc 文件的末尾，这样每次登录时就会自动加载环境变量，无需手动执行。

第 2 步：准备 ONNX 模型

首先，您需要将您的 .onnx 模型文件上传到您的 Atlas 开发板上，例如上传到 ~ (root的主目录) 下。

假设您有一个名为 my_model.onnx 的模型。

第 3 步：确定关键参数

atc 命令需要一些关键参数才能正确转换模型。

  1. SoC 版本 (--soc_version): 这是最重要的参数之一，它告诉 ATC 为哪款具体的芯片进行编译。

    • 从您之前的 npu-smi 输出中，我们可以看到 NPU 型号是 310B4。

    • 因此，对应的 SoC 版本就是 Ascend310B4。

  2. 输入格式与维度 (--input_shape): 您必须明确告诉 ATC 模型的输入张量（Tensor）的形状。

    • 格式通常是 N,C,H,W (批量, 通道, 高度, 宽度)。

    • 例如，一个处理 224x224 的3通道彩色图片，批量为1的模型，其 input_shape 就是 1,3,224,224。

    • 注意：您需要知道您的 ONNX 模型的输入节点名称。通常可以用 Netron 这样的工具打开 .onnx 文件查看。

  3. 框架类型 (--framework): 指明原始模型的框架。对于 ONNX，该值为 5。

第 4 步：执行 ATC 转换命令

现在，我们可以组合出一个完整的 atc 命令。

基础转换示例：

假设您的 ONNX 模型 (my_model.onnx) 有一个名为 input_tensor 的输入节点，期望的输入形状是 1,3,224,224。

bash
复制
atc --model=./my_model.onnx \ --framework=5 \ --output=./my_model_output \ --soc_version=Ascend310B4 \ --input_shape="input_tensor:1,3,224,224"

命令解析:

  • --model=./my_model.onnx: 指定输入的 ONNX 模型文件路径。

  • --framework=5: 5 代表 ONNX 框架。

  • --output=./my_model_output: 指定输出路径和文件名前缀。ATC 会自动生成一个名为 my_model_output.om 的文件。

  • --soc_version=Ascend310B4: 关键！ 指定目标芯片为 Ascend 310B。

  • --input_shape="input_tensor:1,3,224,224": 关键！ 指定模型输入节点的名称和其形状。

执行后，如果一切顺利，您会看到类似以下的成功信息：

复制
ATC run success

然后您可以通过 ls 命令看到当前目录下生成了 my_model_output.om 文件。

高级选项与常见问题

  1. 动态 Batch Size / 动态分辨率

    如果您希望模型能处理变化的批量大小或图像尺寸，可以使用动态参数。例如，支持 batch size 从1到8：

bash
复制
atc --model=./my_model.onnx \ --framework=5 \ --output=./my_model_dynamic \ --soc_version=Ascend310B4 \ --input_shape="input_tensor:-1,3,224,224" \ --dynamic_batch_size="1,2,4,8"

  • input_shape 中的 -1 表示该维度是动态的。

    * `--dynamic_batch_size` 指定了支持的 batch size 档位。

  1. AIPP (人工智能预处理)

    昇腾芯片支持在硬件上完成数据预处理（如图像缩放、色域转换、减均值/归一化等），这被称为 AIPP。这可以大大减轻 CPU 的负担，提升端到端性能。配置 AIPP 需要一个单独的配置文件。

    • 这是一个非常强大的性能优化功能，如果您的应用场景需要，建议查阅官方 AIPP 文档来配置。

  2. 算子不支持问题

    如果转换失败，并提示某个 "Operator" 不支持，意味着您的 ONNX 模型中包含了当前 CANN 版本无法转换的算子。解决方法通常是：

    • 修改模型：尝试用其他支持的算子组合来替换不支持的算子。

    • 升级 CANN 版本：新版本的 CANN 可能会支持更多的算子。

    • 自定义算子：对于高级用户，可以开发自定义算子来弥补支持的不足。

为什么“不需要”？

  • 您已经通过 source /usr/local/Ascend/ascend-toolkit/set_env.sh 命令，在当前的终端会话（也就是这个 (base) 环境中）加载了昇腾 CANN 工具包的环境变量。

  • 避免潜在的冲突：(base) 环境是 Conda 的基础环境，它本身包含了一套独立的 Python 解释器和大量的库。这些库的版本有可能与昇腾工具链所依赖的系统库产生未知的冲突。

  • 简化问题排查：如果 atc 命令在退出 base 环境后仍然报错，那么就可以基本排除是 Conda 环境造成的干扰，从而更容易定位问题的根源。

import onnx import sys def get_model_inputs(model_path): """ 加载ONNX模型并打印其输入节点的名称和形状。 """ try: # 1. 加载模型 model = onnx.load(model_path) # 2. 获取模型的输入信息 (model.graph.input 是一个列表) inputs = model.graph.input if not inputs: print(f"错误: 在模型 '{model_path}' 中没有找到输入节点。") return print(f"'{model_path}' 的输入信息:") print("-" * 30) # 3. 遍历所有输入节点 (通常只有一个) for i, input_node in enumerate(inputs): # 获取输入名称 name = input_node.name # 获取输入的形状信息 shape_proto = input_node.type.tensor_type.shape shape = [] for dim in shape_proto.dim: # 维度可能是数字 (dim_value) 或动态的名称 (dim_param) if dim.HasField('dim_value'): shape.append(dim.dim_value) elif dim.HasField('dim_param'): # 对于动态维度，我们用问号'?'或名称本身表示 shape.append(f"'{dim.dim_param}'(动态)") else: shape.append('?') print(f" 输入节点 {i+1}:") print(f" -> 名称 (Name): {name}") print(f" -> 形状 (Shape): {shape}") print("-" * 30) except Exception as e: print(f"处理模型 '{model_path}' 时发生错误: {e}") if __name__ == "__main__": # 从命令行参数获取模型文件名 if len(sys.argv) < 2: print("用法: python3 inspect_onnx.py <你的模型.onnx>") sys.exit(1) model_file = sys.argv[1] get_model_inputs(model_file)

粘贴完成后，按 Esc 键退出插入模式，然后输入 :wq 并回车，即可保存并退出 vim。

这些警告信息是什么意思？

让我们来分解一下这些无害的“系统通知”：

  1. UserWarning: A NumPy version ... is required ... (detected version ...)

    • 含义：这是最常见的一种警告。它表示昇腾工具链内部使用的某个科学计算库 (SciPy) 是为某个特定范围的 NumPy 版本设计的，而您系统中的 NumPy 版本稍新一些。

    • 影响：通常没有影响。这就像一个软件说“我最喜欢和 Windows 10 一起工作”，但它在 Windows 11 上也能正常运行。只要后续不出现真正的错误，就可以完全忽略它。

  2. ImportWarning: ... find_spec() not found; falling back to find_module()

    • 含义：这是一个非常底层的 Python 内部警告。它在说“我用新方法加载模块失败了，但我会自动切换回老方法，并且成功了”。

    • 影响：完全没有影响。这是工具内部自己的处理机制，对用户来说是透明的。

  3. SyntaxWarning: "is not" with a literal. Did you mean "!="?

    • 含义：这是一个给昇腾软件开发者的代码风格建议。它在提示他们有一行代码可以写得更规范。

    • 影响：对您来说完全没有影响。这不会改变程序的任何功能。

好的，将 YOLOv8 的推理代码从使用 ultralytics (.pt) 迁移到昇腾 NPU (.om) 是一个常见的需求，这能极大地提升在 Atlas 开发板上的运行速度。

这个修改的核心是用华为的 aclruntime 库替换 ultralytics.YOLO 库。aclruntime 是 pyACL (Python Ascend Computing Language) 的一部分，专门用于在昇腾硬件上加载并执行 .om 模型。

这涉及到几个关键步骤：

  1. 加载 OM 模型: 使用 aclruntime 加载 yolo_best_640.om 文件。

  2. 图像预处理 (Pre-processing): OpenCV 读取的图像是 HWC、BGR 格式的 uint8 数据。而 YOLO 模型需要的是 CHW、RGB 格式的 float32 数据，并且尺寸要严格匹配模型（例如 640x640）。这一步必须在 NPU 推理前完成。

  3. 执行推理 (Inference): 将预处理后的数据送入 NPU 计算。

  4. 结果后处理 (Post-processing): NPU 的原始输出是巨大的数字张量 (Tensor)，需要对其进行解析，通过置信度过滤和非极大值抑制 (NMS) 来得到最终的边界框、类别和分数。

# -*- coding: utf-8 -*- import cv2 import os import numpy as np import multiprocessing as mp import time from queue import Empty from aclruntime import AclLiteModel # 导入昇腾推理库 # --- 类别名称定义 --- # OM模型本身不包含类别名称，需要我们手动提供。 # 请根据您模型训练的类别进行修改。 CLASS_NAMES = [ 'person', 'bicycle', 'car', 'motorcycle', 'airplane', 'bus', 'train', 'truck', 'boat', 'traffic light', 'fire hydrant', 'stop sign', 'parking meter', 'bench', 'bird', 'cat', 'dog', 'horse', 'sheep', 'cow', 'elephant', 'bear', 'zebra', 'giraffe', 'backpack', 'umbrella', 'handbag', 'tie', 'suitcase', 'frisbee', 'skis', 'snowboard', 'sports ball', 'kite', 'baseball bat', 'baseball glove', 'skateboard', 'surfboard', 'tennis racket', 'bottle', 'wine glass', 'cup', 'fork', 'knife', 'spoon', 'bowl', 'banana', 'apple', 'sandwich', 'orange', 'broccoli', 'carrot', 'hot dog', 'pizza', 'donut', 'cake', 'chair', 'couch', 'potted plant', 'bed', 'dining table', 'toilet', 'tv', 'laptop', 'mouse', 'remote', 'keyboard', 'cell phone', 'microwave', 'oven', 'toaster', 'sink', 'refrigerator', 'book', 'clock', 'vase', 'scissors', 'teddy bear', 'hair drier', 'toothbrush' ] # COCO 80 classes example class AscendYolo: """ 用于封装昇腾NPU模型推理的辅助类 """ def __init__(self, model_path, model_width=640, model_height=640, conf_thres=0.4, iou_thres=0.5): self.model_path = model_path self.model_width = model_width self.model_height = model_height self.conf_threshold = conf_thres self.iou_threshold = iou_thres print(f"[AscendYOLO] 正在加载模型: {model_path}") self.model = AclLiteModel(model_path) print(f"[AscendYOLO] ✅ 模型加载成功。") def preprocess(self, frame): """图像预处理：resize, BGR->RGB, HWC->CHW, uint8->float32""" h, w, _ = frame.shape # 1. Resize并填充以保持宽高比 scale = min(self.model_width / w, self.model_height / h) resized_w, resized_h = int(w * scale), int(h * scale) resized_frame = cv2.resize(frame, (resized_w, resized_h)) # 2. 创建一个灰色的画布 padded_frame = np.full((self.model_height, self.model_width, 3), 114, dtype=np.uint8) # 3. 将resize后的图像粘贴到画布中央 top = (self.model_height - resized_h) // 2 left = (self.model_width - resized_w) // 2 padded_frame[top:top+resized_h, left:left+resized_w] = resized_frame # 4. BGR -> RGB, HWC -> CHW, to float32 rgb_frame = cv2.cvtColor(padded_frame, cv2.COLOR_BGR2RGB) transposed_frame = rgb_frame.transpose((2, 0, 1)) input_tensor = np.ascontiguousarray(transposed_frame, dtype=np.float32) / 255.0 return input_tensor[np.newaxis, :], (h, w), (scale, left, top) def inference(self, input_tensor): """执行推理""" result_list = self.model.execute([input_tensor]) return result_list[0] def postprocess(self, raw_output, original_shape, transform_params): """结果后处理：解析输出，NMS，缩放边界框""" detections = [] # raw_output shape is (1, 84, 8400) for YOLOv8 # (cx, cy, w, h, cls_conf...) output_data = raw_output[0].T # Transpose to (8400, 84) scores = np.max(output_data[:, 4:], axis=1) # 快速过滤掉大部分低置信度的框 valid_indices = scores > self.conf_threshold box_data = output_data[valid_indices] scores = scores[valid_indices] class_ids = np.argmax(box_data[:, 4:], axis=1) # 坐标转换 cxcywh -> xyxy boxes = self._cxcywh_to_xyxy(box_data[:, :4]) # NMS indices = cv2.dnn.NMSBoxes(boxes.tolist(), scores.tolist(), self.conf_threshold, self.iou_threshold) orig_h, orig_w = original_shape scale, pad_left, pad_top = transform_params # 缩放回原始图像尺寸 for i in indices: x1, y1, x2, y2 = boxes[i] # 移除填充 x1 = (x1 - pad_left) / scale y1 = (y1 - pad_top) / scale x2 = (x2 - pad_left) / scale y2 = (y2 - pad_top) / scale # 裁剪到图像边界 x1 = max(0, x1) y1 = max(0, y1) x2 = min(orig_w, x2) y2 = min(orig_h, y2) detections.append([int(x1), int(y1), int(x2), int(y2), float(scores[i]), int(class_ids[i])]) return detections def _cxcywh_to_xyxy(self, x): y = np.copy(x) y[:, 0] = x[:, 0] - x[:, 2] / 2 # top left x y[:, 1] = x[:, 1] - x[:, 3] / 2 # top left y y[:, 2] = x[:, 0] + x[:, 2] / 2 # bottom right x y[:, 3] = x[:, 1] + x[:, 3] / 2 # bottom right y return y def frame_producer(q_in): """进程1: 帧生产者 (与原版相同)""" stream_url = "http://192.168.5.100:8080/stream" cap = cv2.VideoCapture(stream_url) if not cap.isOpened(): print("[Producer] ❌ 错误: 无法打开视频流。") return print("[Producer] ✅ 视频流连接成功。") while True: ret, frame = cap.read() if not ret: print("[Producer] 🟡 视频流中断，正在尝试重连...") time.sleep(1) cap.release() cap = cv2.VideoCapture(stream_url) continue try: q_in.get_nowait() except Empty: pass q_in.put(frame) def inference_consumer(q_in, q_out): """进程2: 推理消费者 (已修改为使用昇腾NPU)""" current_dir = os.path.dirname(os.path.abspath(__file__)) colors = [ (255, 0, 0), (0, 255, 0), (0, 0, 255), (255, 255, 0), (0, 255, 255), (255, 0, 255), (192, 192, 192), (128, 0, 128) ] model_paths = [f for f in os.listdir(current_dir) if f.endswith('.om')] # 对帧进行推理和绘制 for model_name, model_info in models.items(): model, color = model_info["model"], model_info["color"] # 1. 预处理 input_tensor, original_shape, transform_params = model.preprocess(frame) # 2. 推理 raw_output = model.inference(input_tensor) # 3. 后处理 detections = model.postprocess(raw_output, original_shape, transform_params) for (x1, y1, x2, y2, conf, cls_id) in detections: class_name = CLASS_NAMES[cls_id] if cls_id < len(CLASS_NAMES) else 'Unknown' label = f'{os.path.splitext(model_name)[0]}: {class_name} {conf:.2f}' cv2.rectangle(frame, (x1, y1), (x2, y2), color, 2) cv2.rectangle(frame, (x1, y1 - 20), (x1 + len(label) * 10, y1), color, -1) cv2.putText(frame, label, (x1, y1 - 5), cv2.FONT_HERSHEY_SIMPLEX, 0.5, (255, 255, 255), 2) # 绘制图例 legend_y = 20 for model_name, model_info in models.items(): legend_text = f"- {os.path.splitext(model_name)[0]}" cv2.putText(frame, legend_text, (10, legend_y), cv2.FONT_HERSHEY_SIMPLEX, 0.6, (0, 0, 0), 3) cv2.putText(frame, legend_text, (10, legend_y), cv2.FONT_HERSHEY_SIMPLEX, 0.6, model_info["color"], 2) legend_y += 25 if q_out.full(): q_out.get_nowait() q_out.put(frame) if __name__ == "__main__": mp.set_start_method('spawn', force=True) input_queue = mp.Queue(maxsize=1) output_queue = mp.Queue(maxsize=1) producer_process = mp.Process(target=frame_producer, args=(input_queue,)) consumer_process = mp.Process(target=inference_consumer, args=(input_queue, output_queue,)) producer_process.start() consumer_process.start() print("[Main] ✅ 主程序启动，等待处理后的帧...") print("[Main] 按 'q' 键退出。") while True: try: frame = output_queue.get(timeout=5) cv2.imshow('Live Multi-Model Detection (Ascend NPU)', frame) if cv2.waitKey(1) & 0xFF == ord('q'): break except Empty: if not consumer_process.is_alive() or not producer_process.is_alive(): print("[Main] 🟡 一个或多个子进程已停止，程序退出。") break print("[Main] 🟡 等待处理结果超时...") continue print("[Main] 正在关闭...") producer_process.terminate() consumer_process.terminate() producer_process.join() consumer_process.join() cv2.destroyAllWindows()
